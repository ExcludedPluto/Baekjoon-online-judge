알고리즘
	1. 위상정렬 시행 후, 모든 가수들이 정렬한거에 들어와있는지 체크. 안들어와있으면 0 출력하면 됨


푼 과정
	1. 위상정렬까진 알아서 일단 위상정렬 구현함
	2. 사이클 처리에 대해 생각함. -> 사이클? unionFind 인가? -> 아니었음. 이미 같은 set에 들어와있는 두 점이 같은 순서로 또 들어올 수도 있었음 -> 사이클이 아닌데 0출력해버림
	3. A -> B -> C -> A 일때 사이클 발생이므로 저 세 노드는 위상정렬시 영원히 beforeCount가 0이 되지 않음을 알아챔
	4. 위상정렬 후 모든 노드들이 정렬되어있는 확인하고, 아니면 0출력하는 식으로 하니 통과함

	-> 하지만 내 위상정렬 방식은 시간이 많이 걸리는 듯.
	-> 다른 방법을 찾아보자
	-> list 안쓰고 order 정렬들어가는 배열에 바로 start와 last로 접근하여 위상정렬을 실행하였는데도 시간은 큰차이 안났음
	-> 시간차는 없지만 더 간단하므로 일단 다음부터 이걸로 하자.
